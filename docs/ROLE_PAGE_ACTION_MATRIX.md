# 角色 - 页面 - 动作对照表

这份表用于快速查看当前 MDAMS Prototype 里，不同角色大致能看到哪些页面、能执行哪些动作。  
它是基于当前 `RBAC + scope` 权限骨架整理的，后续如果权限再细化，这份表也会同步更新。

## 1. 角色说明

- `system_admin`：系统管理员
- `resource_user`：数据浏览与申请者
- `collection_owner`：藏品管理者
- `application_reviewer`：申请审批人
- `three_d_operator` / `three_d_resource_manager`：三维数据管理角色

## 4. scope 说明

当前系统不只是“按角色控制”，还需要按资源范围控制。  
因此同一个角色能不能操作，除了看角色本身，还要看资源是否落在 scope 内。

当前主要 scope 规则：

- `open`：馆内开放资源，允许有浏览权限的用户查看
- `owner_only`：仅责任范围内角色可见
- `own_scope`：仅当前用户负责的藏品范围可操作

典型判断方式：

- `resource_user`：只能看 `open`
- `collection_owner`：可以看 `open`，也可以看自己责任范围内的 `owner_only`
- `application_reviewer`：可以看申请相关对象，但不一定可以改资源元数据
- `system_admin`：通常拥有全局可见和全局管理权限

## 5. 使用建议

这张表适合用于：

- 你自己做功能验证时逐项对照
- 新功能上线前检查菜单和按钮是否正确
- 后续补权限测试时作为基线
- 跟团队对齐“谁能看什么、谁能做什么”

如果以后角色继续细分，可以在这份表上继续加列，不需要推翻重来。
