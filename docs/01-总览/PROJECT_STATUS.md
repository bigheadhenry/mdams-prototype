# MDAMS 项目状态

- 最后核对日期：2026-04-06
- 核对范围：`README.md`、`docker-compose.yml`、`backend/app/main.py`、`backend/app/routers/`、`frontend/src/App.tsx`、`backend/tests/`、`frontend/tests/`
- 当前阶段：可持续开发与演示阶段

## 1. 当前定位

MDAMS Prototype 当前最准确的定位不是“二维影像上传 PoC”，而是一个已经具备多条核心业务链路的馆内数字资源管理原型底座。

当前已形成的稳定板块包括：

- 二维影像资产与 IIIF 预览
- 图像记录 `ImageRecord` 工作流
- 三维资源对象与多文件资源包管理
- 统一平台目录与统一详情
- 登录、角色、权限与责任范围控制
- 资源申请、审批与交付导出
- 基础测试体系与工作日志

## 2. 当前已实现范围

### 2.1 二维影像

当前二维相关实现已经覆盖：

- 资产上传、列表、详情与删除
- 预览图生成与列表缩略图展示
- IIIF Manifest 生成
- Mirador 预览
- 原文件下载与 BagIt 下载
- 衍生文件策略与 IIIF access 处理

### 2.2 图像记录工作流

当前已具备基于 `ImageRecord` 的角色拆分工作流：

- 录入人员创建和编辑图像记录
- 图像记录提交与退回
- 摄影上传人员查看待上传池
- 文件上传与记录匹配
- 绑定后基础校验与重复检测

这意味着当前二维链路已经不再只是“文件中心”，而是开始支持“元数据记录”和“影像文件”分离协作。

### 2.3 资源申请与交付

当前已具备：

- 申请车
- 申请单创建
- 审批通过 / 拒绝
- 导出交付包

相关接口已经在后端独立成 `applications` 路由。

### 2.4 三维子系统

当前三维相关实现已经覆盖：

- 三维资源对象与版本信息
- 多文件上传
- 模型 / 点云 / 倾斜摄影角色区分
- 对象详情与查看摘要
- Web 展示链路
- 三维详情构造与字典服务

### 2.5 统一平台

当前统一平台相关实现已经覆盖：

- 来源注册表
- 来源摘要接口
- 统一资源目录
- 统一资源详情
- 按关键条件筛选资源

当前已知来源至少包括二维与三维两类适配器。

### 2.6 认证、权限与范围

当前权限框架已经不是占位实现，而是实际参与前后端行为控制：

- 登录、会话、上下文接口可用
- 内置测试用户会自动播种
- 后端基于 `ROLE_PERMISSIONS` 做权限判断
- 前端菜单按权限可见
- `collection_owner` 支持基于 `collection_scope` 的责任范围过滤
- `owner_only` 可见范围已进入后端访问判断

### 2.7 测试

当前测试体系已经覆盖：

- 后端单元 / 契约 / 集成 / smoke / subsystem 测试
- 前端 Playwright 菜单、统一平台、图像记录相关回归
- 健康检查、配置、权限、平台目录、三维链路等关键模块

## 3. 当前真实边界

虽然当前功能已经明显超出早期 PoC，但项目仍然属于原型阶段，主要边界包括：

- 权限模型已可用，但还不是生产级身份治理
- 平台聚合已成立，但统一检索与更多来源接入仍需继续扩展
- 三维链路已打通，但规范化和兼容性仍需继续加强
- 长期保存、治理、审计、迁移与监控能力还不完整
- 文档体系仍在更新中，部分历史文档尚未完全替换

## 4. 当前代码结构信号

根据当前代码结构，项目主能力已经比较清晰：

- 后端主路由包含 `auth`、`assets`、`applications`、`downloads`、`health`、`iiif`、`ingest`、`image-records`、`platform`、`three-d`、`ai`
- 前端主页面包含仪表盘、二维资源、申请车、入库处理、统一平台、三维管理、申请管理、图像记录工作台
- 部分历史文档仍在描述旧的 `MEAM` 阶段命名或旧部署方式，不应再作为当前事实来源

## 5. 当前阶段判断

当前最准确的判断是：

> MDAMS 已经进入“主链路可用、模块边界清晰、可继续平台化”的阶段。

这意味着它已经具备：

- 演示能力
- 继续开发能力
- 真实模块边界
- 基础权限语义
- 可验证的测试支撑

但它仍未进入：

- 完整产品化阶段
- 完整治理阶段
- 完整标准化与长期保存阶段

## 6. 下一步优先方向

建议后续优先继续推进以下方向：

1. 收敛正式文档入口，消除历史重复文档
2. 补齐权限、平台、图像记录、三维子系统专题文档
3. 继续稳定统一平台与跨来源聚合能力
4. 继续完善角色范围、审批规则与治理能力
5. 扩展测试矩阵和回归覆盖面
