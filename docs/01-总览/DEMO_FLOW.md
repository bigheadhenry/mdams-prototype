# 标准演示流程

## 1. 演示目的

这份文档用于统一 MDAMS Prototype 的演示路径，让开发、联调、汇报和对外展示都尽量走同一条链路。

当前演示重点不只是“上传一张图”，而是要展示：

- 登录与角色差异
- 二维影像主链路
- IIIF 与 Mirador 预览
- 利用申请与交付
- 三维对象管理与查看
- 统一平台目录与统一详情

## 2. 演示前提

演示前建议确认：

- 前后端服务已启动
- Cantaloupe 可访问
- 数据库和 Redis 正常
- 至少准备一批二维样本资源
- 至少准备一个 ready 的三维展示版

建议先检查：

- `docker compose ps`
- `http://localhost:8000/health`
- `http://localhost:8000/ready`

## 3. 二维演示主流程

### 步骤 1：登录

先用馆内用户登录。不同角色会看到不同菜单和不同资源范围。

### 步骤 2：查看二维资源列表

进入二维资源页面，观察：

- 资源是否能正常列出
- 可见范围是否正确
- 资源状态是否正确

### 步骤 3：打开资源详情

检查详情页是否能看到：

- 基础信息
- 技术元数据
- 可见范围
- 来源 / 状态信息

### 步骤 4：打开 Mirador

在 ready 资源上打开 Mirador，检查：

- Manifest 是否可用
- 图像能否正常加载
- 能否进行放大、缩小和浏览

### 步骤 5：加入申请单

在 Mirador 或资源入口中加入申请车，检查：

- 是否能加入申请单
- 是否能填写用途和备注
- 是否能提交正式申请

### 步骤 6：审批与交付

进入申请管理页，检查：

- 申请单是否可见
- 是否能审批通过或拒绝
- 审批通过后是否能导出交付包

## 4. 三维演示主流程

### 步骤 1：进入三维管理页

检查：

- 对象级聚合是否正常
- 版本列表是否正常
- Web 展示版是否清晰可见

### 步骤 2：打开三维 viewer

检查：

- `web_preview_status = ready` 的版本能否正常展示
- 模型是否能交互查看
- 文件构成是否正确回显

### 步骤 3：查看对象关联

检查三维对象与藏品对象的关联是否清楚。

## 5. 统一平台演示主流程

进入统一资源目录后，检查：

- 是否能看到二维和三维来源
- 是否能按 profile、类型、状态筛选
- 是否能打开统一详情
- 是否能跳回来源详情或预览

## 6. 角色演示建议

建议至少演示三类账号：

- `system_admin`
- `resource_user`
- `collection_owner`

这样可以清楚展示：

- 菜单是否裁剪
- 资源是否受 scope 控制
- 申请和审批是否分离

## 7. 演示结束标准

当以下内容都成立时，可以认为演示成功：

- 登录正常
- 二维资源可见
- Mirador 可用
- 申请车可用
- 申请管理可用
- 三维 viewer 可用
- 统一平台可用
- 不同角色看到的内容不同
