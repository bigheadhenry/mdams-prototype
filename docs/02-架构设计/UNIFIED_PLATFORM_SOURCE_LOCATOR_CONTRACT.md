# 统一平台来源定位契约

- 最后核对日期：2026-04-22
- 适用范围：统一平台目录、统一平台详情、Mirador AI 候选、申请车、平台适配器输出

## 1. 契约目标

统一平台不再把“一个字符串同时承担唯一标识和可解析语义”作为主表达方式。  
项目当前采用的稳定做法是：

- `id` 作为平台层唯一标识
- `source_system` + `source_id` 作为显式来源定位
- `resource_id` 仅作为历史兼容读入，不进入新契约和新展示

## 2. 主契约字段

### 2.1 平台层主键

| 字段 | 角色 | 说明 |
|---|---|---|
| `id` | 平台唯一标识 | 平台层统一资源 ID，供目录、详情和关联跳转使用 |
| `source_system` | 来源命名空间 | 例如 `image_2d`、`three_d` |
| `source_id` | 来源内主键 | 来源系统内部稳定 ID |

### 2.2 展示与跳转字段

| 字段 | 角色 | 说明 |
|---|---|---|
| `detail_url` | 平台详情入口 | `GET /api/platform/resources/{source_system}/{source_id}` |
| `manifest_url` | 访问表现入口 | 二维用于 IIIF Manifest，三维用于来源详情或预览入口 |
| `source_detail_url` | 来源详情入口 | 指向来源系统原生详情 |
| `actions` | 共享动作列表 | 统一展示预览、详情、下载、导出等动作 |

### 2.3 兼容字段

| 字段 | 状态 | 说明 |
|---|---|---|
| `resource_id` | 兼容读入 | 仅用于历史数据、旧路由或旧样例读取 |
| `current_resource_id` | 兼容上下文 | 仅用于过渡性的旧请求体，不作为新字段传播 |

## 3. 路由规则

### 3.1 主路由

- `GET /api/platform/resources/{source_system}/{source_id}`

该路由是统一平台详情的唯一主入口。  
路由层和适配器层都应优先按显式来源定位解析。

### 3.2 兼容路由

- `GET /api/platform/resources/{resource_id}`

该路由仅作为历史兼容入口保留，并标记为弃用。  
兼容路由只允许读取，不应出现在新文档中的推荐路径中。

## 4. 展示规则

- 目录、详情、Mirador 候选和申请车条目都优先显示 `source_system/source_id`
- 不再把 `resource_id` 作为默认 UI 文案
- 若需要历史追踪，`resource_id` 可以在调试或兼容层保留，但不进入用户主视图

## 5. 适配器规则

- 每个来源适配器都应返回 `source_system`、`source_label`、`source_id`
- 平台层不负责推断 `source_id` 的语义，只负责把来源定位传递给对应适配器
- 新增来源时，不允许回到“用冒号拼接后再拆分”的主契约模式

## 6. 迁移原则

1. 新响应只输出显式来源定位字段。
2. 旧字段可以读，不建议再写入新样例。
3. 旧路由保留为兼容层，并逐步降级为弃用入口。
4. 文档、测试样例、前端展示应尽量同步到同一主契约。

## 7. 当前结论

当前项目已经完成从 `resource_id` 主导到 `source_system/source_id` 主导的迁移。  
后续所有统一平台扩展都应以本契约为准。
