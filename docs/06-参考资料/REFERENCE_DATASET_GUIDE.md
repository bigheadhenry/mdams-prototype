# 参考数据与资源包说明

- 最后核对日期：2026-04-06
- 核对范围：`reference/`、`backend/scripts/`

## 1. 目标

本文件用于说明仓库中的 `reference/` 目录目前承载什么，以及它和导入脚本、参考映射文档之间的关系。

## 2. 当前目录用途

`reference/` 当前主要不是运行时依赖，而是参考样例与导入素材目录。

它主要包含：

- 参考资源包
- 样例图片
- 元数据表格

## 3. 当前已知内容

当前目录中可见的重点内容包括：

- `资源包/`
- `digicol-reference-1.jpg`
- `影像系统描述和管理元数据架构.xlsx`

其中 `资源包/` 下又包含多类资源子目录，例如：

- 业务活动影像
- 其他影像
- 古树影像
- 文物建筑影像
- 文物影像
- 考古影像

## 4. 当前作用

这些参考数据当前主要服务以下用途：

1. 参考导入
2. 元数据映射验证
3. 二维资源样例构造
4. 工作流与平台演示

## 5. 与脚本的关系

当前 `backend/scripts/` 下多份脚本与 `reference/` 有直接关系，尤其是：

- 生成参考 manifest
- 导入参考资源
- 校验导入结果
- 输出完整性报告

因此，`reference/` 更适合被视为：

> 当前项目的参考样例数据源，而不是普通静态附件目录。

## 6. 使用建议

- 不要随意改动样例资源结构
- 导入前优先先做 dry-run
- 如果新增新的参考目录，最好同步更新导入与映射文档
