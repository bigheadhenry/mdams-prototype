# 前端菜单可见矩阵

- 最后核对日期：2026-04-06
- 核对范围：`frontend/src/App.tsx`、`frontend/src/auth/permissions.ts`、`backend/app/permissions.py`

## 1. 目标

本文件用于说明当前前端主菜单的显示规则，以及菜单与权限、角色之间的对应关系。

当前结论以代码为准。

## 5. 需要特别说明的菜单

### 5.1 `6` 统一资源详情

这个菜单并不代表用户会直接从左侧导航跳到一个通用空详情页。它主要是为了在平台目录内部切换详情视图时，维持当前页面状态。

因此：

- 只要用户具备 `platform.view`，这个菜单 key 就会被视为可用
- 真正能不能看到详情，还取决于是否已经选中了统一资源

### 5.2 `4` 入库处理

当前这个菜单不是“所有二维相关人员都能看到”，而是只对具备以下任一能力的角色开放：

- 上传
- 入库审核
- 二维编辑

这意味着图像记录录入人员 `image_metadata_entry` 当前不会看到该入口，因为它的工作台已经独立放到菜单 `9`。

### 5.3 `9` 图像记录工作台

当前这个入口服务两类角色：

- `image_metadata_entry`
- `image_photographer_upload`

两者进入同一工作台组件，但实际能看到的列表、操作按钮和业务入口不同。

## 6. 当前实现原则

当前菜单实现遵循以下原则：

1. 先按权限决定菜单是否可见
2. 进入页面后再按权限控制页面内部动作
3. 后端接口继续做最终兜底

也就是说：

- 菜单可见不等于所有按钮都可用
- 菜单隐藏也不能替代后端权限校验
