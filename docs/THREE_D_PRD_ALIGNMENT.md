# 三维资源管理系统 PRD 对照表

本文档用于对照 `C:\Users\bighe\OneDrive\AI\Codex\3D-DAMS\prd.md` 的要求，梳理当前 MDAMS 仓库中三维子系统的完成度、对齐项与后续缺口。

## 一、总体结论

当前三维子系统已经达到“可管理、可分版本、可资源包化、可接入统一平台”的原型阶段，方向与 PRD 基本一致，但还没有完全落到“藏品对象 - 三维对象 - 文件对象 - 长期保存”这一整套生产级结构。

已经完成的内容更偏向“数字对象管理原型”，尚未完成的部分主要集中在：
- 与藏品库的强关联
- 完整的生产链路记录
- 长期保存与迁移对接
- 真正的在线 Web 3D 展示与治理能力

## 三、按 PRD 视角的系统分层

### 3.1 当前实现接近的层次

- 藏品对象：已有概念支撑，但未正式与藏品库强绑定。
- 三维对象：已基本完成，是当前管理核心。
- 文件对象：已完成，支持模型、点云、倾斜摄影图像多文件。

### 3.2 PRD 要求中的四类数据

| 数据类别 | PRD 定义 | 当前实现情况 |
|---|---|---|
| 原始采集数据 | 原始照片、点云、扫描、说明 | 已有文件接入，但采集记录不完整。 |
| 处理数据 | 高精度模型、低精度模型、纹理、材质 | 已有版本和文件角色，但处理过程记录仍弱。 |
| 发布数据 | Web 展示模型、轻量化模型 | 已有 `web_preview_status`，但 Web 展示链路仍是原型。 |
| 文档资料 | 采集说明、处理说明、质检、技术报告 | 已有文档层概念，但还没有完整档案化。 |

## 四、当前实现与 PRD 的关键差距

1. 当前系统更像“数字对象管理系统”，还不是完整的“藏品数字三维生产与保存系统”。
2. 版本与 Web 展示状态已经做出来了，但发布流程、审批流程、回退流程还没有补全。
3. 对象级聚合已经有了，但缺少对象级治理字段，例如采集批次、关联藏品、责任人、保存级别。
4. 长期保存体系是 PRD 中很重要的一层，目前只停留在资源包和下载层，没有完成归档体系。

## 五、推荐的三条推进方向

### 方向一：先把数据模型彻底对齐 PRD

建议优先完成：
- 藏品对象与三维对象的正式关联
- 三维对象元数据字典标准化
- 原始 / 处理 / 发布 / 文档四层的字段边界收紧

### 方向二：补全生产链路记录

建议补齐：
- 采集记录
- 处理记录
- 发布记录
- 质检记录

这是把“能管理资源”升级为“能追溯生产过程”的关键一步。

### 方向三：把展示层与保存层分开做实

建议分别推进：
- Web 三维查看器
- 轻量化发布模型
- 长期保存包
- 保存校验与迁移接口

这样系统才能同时满足“当前展示”和“长期保存”两种要求。

## 六、建议的下一阶段优先级

1. 数据模型与藏品关联对齐
2. 生产链路记录补齐
3. Web 展示与长期保存分层实现

这三步做完，三维子系统就会从“可用原型”进入“可持续平台”的阶段。
