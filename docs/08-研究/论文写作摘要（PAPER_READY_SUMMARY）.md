# 论文写作摘要

## 用途

本文档提供一版可直接复用到论文、研究报告或答辩材料中的压缩摘要。

它不替代完整研究文档，而是把当前最稳定、最可写、最不容易漂移的论点先收束出来。

## 一句话定位

MDAMS Prototype 是一个面向博物馆/展陈数字资源场景、以数字资产为核心、具有保存意识并采用选择性标准对齐策略的研究型技术原型。

## 问题陈述

博物馆与展陈数字资源管理常常处于两种不理想状态之间：一种是停留在文件上传下载层面，另一种是试图从一开始就覆盖完整生产级 DAMS 或藏品管理体系。前者难以支撑可持续管理与研究表达，后者又在原型阶段过于庞杂。

MDAMS 的目标不是覆盖全部功能，而是在有限范围内证明：一个围绕数字资产组织的原型，能够把采集、校验、处理、访问、导出以及部分协作与平台功能组织成一个可演示、可解释、可评估的系统。

## 核心论点

### 1. 以数字资产为核心对象
MDAMS 不把孤立文件当作系统中心，也不试图直接等同于完整馆藏管理体系，而是把数字资产视为连接文件、元数据、处理状态、访问表示和导出表示的核心对象。

### 2. 优先稳定可演示工作流
原型的价值不主要来自功能数量，而来自稳定、可重复、可解释的工作流。当前 MDAMS 最强的工作流包括二维数字资产主链路、图像记录协作链路，以及三维对象与统一平台链路。

### 3. 采用选择性标准对齐
MDAMS 不宣称对所有相关标准实现全面合规，而是采用分层策略：IIIF 和 BagIt 作为直接实现层，OAIS 作为概念框架层，PREMIS 与 NISO Z39.87 作为部分对齐层，CS3DP 作为三维扩展参考层。

### 4. 强调 preservation-aware，而非 fully preservation-compliant
当前系统已经体现出 fixity、处理过程、访问表示、导出包、生命周期意识和部分日志痕迹，因此可以被合理描述为 preservation-aware 原型；但它仍未形成完整的 OAIS、PREMIS 或长期保存体系。

## 当前实现证据

截至 **2026-04-08**，项目已具备以下真实实现证据：
- 二维影像资产上传、列表、详情、预览、IIIF Manifest、Mirador、BagIt 导出；
- `ImageRecord` 角色拆分工作流与绑定/替换机制；
- 平台来源适配器、统一目录与统一详情；
- 三维对象、版本、多文件资源包、viewer 契约与平台接入；
- 登录、角色权限、责任范围过滤；
- 申请、审批与交付包导出；
- Docker Compose、PostgreSQL、Cantaloupe、Celery、Redis 等真实运行基础；
- 后端 `pytest` 与前端 `Playwright` 回归测试。

## 研究贡献候选

1. 提出一个以数字资产为核心的博物馆数字资源管理研究型原型框架；
2. 展示如何在同一原型中整合二维影像、图像记录协作、统一平台和三维资源；
3. 给出一种“选择性标准对齐”的实现与论证路径；
4. 提供一个兼具工程实现证据与研究可解释性的案例。

## 边界与局限

当前系统仍是研究型原型，而不是完整生产系统。

其主要局限包括：
- 对象模型仍需进一步收口；
- PREMIS 风格事件模型尚未 formalize；
- 图像与三维元数据 profile 仍需系统化；
- 治理、审计、迁移与长期保存层仍不完整；
- 更复杂的统一检索、来源接入与审批治理仍在后续阶段。

## 可直接复用的结论段

MDAMS Prototype 的价值不在于以原型身份模拟一个功能完备的生产级 DAMS，而在于通过有限但扎实的实现，证明一个面向博物馆/展陈数字资源场景的数字资产管理原型，能够在真实工程环境中把采集、校验、处理、访问、导出以及部分协作与平台化能力组织为稳定工作流，并通过 IIIF、BagIt、OAIS、PREMIS、NISO Z39.87 和 CS3DP 等标准/框架形成分层、克制而可解释的研究表达。
