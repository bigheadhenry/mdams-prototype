# MDAMS 研究子项目

## 目的

本目录用于支撑 MDAMS Prototype 的并行研究/论文写作工作。

它不是单纯的“项目说明文档”汇总，而是一个与开发主线同步推进的研究工作区，用来持续沉淀：
- 项目事实；
- 研究问题；
- 标准与理论框架；
- 设计决策；
- 评估逻辑；
- 可逐步发展为论文的结构化内容。

## 当前定位

截至 **2026-04-08**，MDAMS 最适合被理解为一个**面向博物馆/展陈数字资源场景的研究型技术原型**，而不是一个已经完全产品化、完全标准化、完全制度化部署的成熟系统。

因此，这条研究线的重点不是夸大其“完整性”，而是要清楚说明：
1. 这个原型回应了什么问题；
2. 它如何把数字资产管理、保存意识、访问互操作与工程实现结合起来；
3. 它如何在二维、图像记录、统一平台、三维资源、申请交付和权限控制之间形成一个可解释的整体；
4. 它如何成为一个既可演示、又可研究、还能继续扩展的系统原型。

## 当前研究主线

目前研究子项目围绕以下几个方向展开：
- 明确 MDAMS 的问题域、对象模型与范围边界；
- 提炼当前仓库中已经存在的“真实项目事实”；
- 将 IIIF、BagIt、OAIS、PREMIS、NISO Z39.87、CS3DP 等标准/框架与项目对应起来；
- 梳理原型的核心概念模型与设计决策；
- 形成后续论文写作可直接复用的结构；
- 持续检查研究文档与当前实现之间是否发生漂移。

## 工作原则

本研究目录遵循以下原则：
- **以当前真实实现为基础**，不把未来设想写成既成事实；
- **区分“已实现”“部分对齐”“概念借鉴”“未来扩展”**；
- **让研究叙述能够反过来服务工程推进**；
- **优先形成可持续扩写的中间文档**，而不是过早追求一次性终稿；
- **持续对齐代码、工作日志、项目状态和研究表达**，减少文档漂移。

## 当前阶段判断

当前最重要的研究任务，不是继续扩大量，而是：
- 让研究文档对齐当前系统事实；
- 把统一对象模型、事件模型和 metadata/profile 从“可讨论”推进到“可复用”，再继续推进到更明确的实施边界；
- 把主链路固化为论文和演示都可复用的案例；
- 继续把现有文档与真实样本、验证和实施边界连接起来；
- 围绕关键讨论问题统一后续决策顺序；
- 先把跨子系统事件边界收口到 detail / test 层；
- 从实施边界清单中挑选 1 到 2 个优先项进入实现或测试补强。
