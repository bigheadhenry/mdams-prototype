# 演示链路论文案例摘要

## 用途

本文档将当前标准化演示主链路压缩为可直接进入论文“当前实现与演示链路”章节的案例摘要。

## 案例 1：二维数字资产主链路

当前 MDAMS 最强的案例链路是二维数字资产主链路。该链路从数字文件进入系统开始，经过资产登记、技术元数据处理、访问副本准备、IIIF Manifest 生成、Mirador 访问和 BagIt 打包输出，构成了一条完整的采集、处理、访问与导出路径。其研究意义在于，它证明系统并非停留于上传下载层面，而是已经能够以数字资产为核心组织多层对象表达，并通过 IIIF 与 BagIt 建立直接的标准实现层。

## 案例 2：图像记录协作链路

图像记录工作流展示了 MDAMS 与普通文件管理工具的重要差异。系统允许元数据录入人员先建立 `ImageRecord`，再由摄影上传人员在待上传池中查看、上传并显式确认绑定或替换文件。该链路证明系统已形成“协作记录对象”与“数字资产对象”相互配合的结构，也说明当前对象模型和 PREMIS 事件模型需要同时覆盖记录提交、退回、绑定和替换等业务动作。

## 案例 3：三维对象与统一平台链路

三维对象链路表明 MDAMS 已不再只是二维图像原型。当前系统能够管理三维对象、版本、多文件资源包和 viewer 预览状态，并通过统一平台把三维资源与二维资源纳入同一聚合视图。该案例的研究意义不在于证明完整 3D 保存体系已经实现，而在于说明项目已经形成 minimum viable 的“对象 + 版本 + 文件包 + 平台接入”结构，支撑其作为多来源数字资源原型的论证。

## 综合结论

这三条案例链共同支撑一个核心判断：MDAMS Prototype 的价值不在于功能数量本身，而在于它已经通过有限但扎实的实现，建立起一组可演示、可解释、可验证的工作流。这些工作流分别承担了主链路证明、协作语义证明和多来源平台化证明，从而使该系统能够被合理解释为一个以数字资产为核心、具有保存意识并采用选择性标准对齐策略的研究型技术原型。
